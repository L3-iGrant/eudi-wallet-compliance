EUDI Wallet Compliance portal is being deployed.

Site: https://{{ .Values.ingress.host }}
Auth: Google SSO, restricted to @{{ .Values.oauth2Proxy.emailDomain }}
      (oauth2-proxy mounted at https://{{ .Values.ingress.host }}/oauth2/)

Before this works you must:

1. Create an OAuth 2.0 Client (Web application) in Google Cloud Console:
   - Authorised JavaScript origin:
       https://{{ .Values.ingress.host }}
   - Authorised redirect URI:
       https://{{ .Values.ingress.host }}/oauth2/callback

2. Create the secret with the client credentials:

   kubectl -n {{ .Release.Namespace }} create secret generic {{ .Values.oauth2Proxy.existingSecret }} \
     --from-literal=client-id='YOUR_CLIENT_ID' \
     --from-literal=client-secret='YOUR_CLIENT_SECRET' \
     --from-literal=cookie-secret="$(openssl rand -base64 32 | tr -- '+/' '-_')"

3. Point a DNS A/CNAME record at the ingress controller:
       {{ .Values.ingress.host }}
