## [1.1.0] (2026-05-05)

### Added

- Support for ISO/IEC mdoc EAA per ETSI TS 119 472-1 §6 (v1.2.1), covering both mDL (ISO/IEC 18013-5) and non-mDL credentials (ISO/IEC 23220).
- 105 mdoc structural and runtime checks across §6.1 - §6.6.
- mdoc parser in `@iwc/shared` using `cbor-x` with inline COSE_Sign1 decoding.
- Five mdoc reference samples (`mdl-eaa-1` through `mdl-eaa-5`) generated by `tools/generate-samples/src/mdoc.ts`.
- Profile-aware upload form with auto-fill on detected payload kind and a profile-mismatch warning when the auto-detected format disagrees with the user's scope.
- Discriminated-union `ReferenceSample` schema (SD-JWT VC + mdoc arms).
- Reference-sample detail page renders the CBOR payload, `docType`, namespace list, decoded protected header, MSO, and namespace contents.

### Changed

- `runAssessment` parses evidence once and filters the registry by `scope.profile ∩ control.profile` in a single intersection check.
- Cross-cutting clause-4 controls switched from the special-case `abstract` profile tag to explicit multi-tagging (`profile: [sd-jwt-vc, mdoc]`); the `abstract` value is removed from the schema.

### Notes

- Cryptographic signature verification for both profiles remains deferred.
- mdoc reference samples carry placeholder signatures; the engine does not verify mdoc signatures.

## [1.0.0] (2026, private)

Initial private release with SD-JWT VC EAA support under ETSI TS 119 472-1 §5 (v1.2.1). The catalogue, engine, runtime status-list resolver, and Next.js self-assessment UI were established in this release.
